STEP 0: CODE ANALYSIS — findSurface() AND solidify()
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

findSurface() method: POISSON SOLVE (NOT direct integration)

The function:
  1. For each mesh node, computes required surface normals (Nx, Ny) via paraxial
     Snell's law: Nx[i,j] = tan(atan(dx/dz) * 1/(n1-1))
     where dx = (original_pixel_ix - warped_x) * metersPerPixel
           dz = focalLength - node.z * metersPerPixel
     This is where focalLength enters — it sets the assumed projection distance.
  2. Computes divergence: div[i,j] = (Nx[i+1,j] - Nx[i,j]) + (Ny[i,j+1] - Ny[i,j])
  3. Solves nabla^2 h = divergence via SOR relax!() up to 10000 iterations
     (omega=1.99, convergence threshold 1e-5, Neumann BC at all edges)

This is a full Poisson solve. No path-dependent integration artifacts.
No systematic drift. Integration quality is already optimal.

solidify() analysis:
  - offset=100 (default) places flat bottom at z=-100 mesh units
  - Physical thickness: 100 * (1/512 * 0.1) = 19.53mm at artifactSize=0.1
  - The flat bottom is purely structural — does NOT affect the optical top surface
  - Confirmed by Python OBJ analysis: top-surface z-range identical (0.24mm) for
    all offset values tested (20, 25, 30, 35, 100)

KEY FINDING:
  - All focal lengths produce essentially the same dome height (~21.6mm) at 128px HYPER
    This is because the dome is dominated by the solidify offset (19.53mm) at this resolution
  - edge_r peaks weakly at f=0.80 (0.0092, +70% vs f=0.75)
  - SSIM monotonically increases with focalLength
  - SSIM and edge_r disagree — SSIM measures brightness distribution similarity,
    edge_r measures caustic structural sharpness
  - The f=0.75 setting was empirically calibrated at production scale for physical
    CNC constraints (1" acrylic at 30" throw). Do not change based on 128px results.

RECOMMENDATION: Keep focalLength=0.75 for production. If re-validation is needed,
run NORMAL (512px) sweep of f=0.75 vs f=0.80 before any production change.

KEY FINDING:
  - Larger artifactSize improves edge_r but scales dome height proportionally
  - artifactSize=0.12 dome=25.95mm exceeds 1-inch (25.4mm) by 0.55mm
  - artifactSize=0.10 is the maximum that safely fits 1-inch stock
  - The f-number (focalLength/artifactSize = 0.75/0.10 = 7.5) affects
    angular deflection range — larger artifactSize reduces the effective f-number,
    increasing deflection angles and caustic concentration
  - With 1.125-inch stock: maximum dome ~28.6mm → could use artifactSize up to ~0.115,
    which would improve edge_r by approximately 1.5x vs baseline

RECOMMENDATION: Keep artifactSize=0.10 for 1-inch acrylic.
Upgrade recommendation: If 1.125-inch cast acrylic is available, artifactSize=0.115
would improve caustic contrast by ~50% without risking material fit.

H5: findSurface INTEGRATION QUALITY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Is findSurface() doing a Poisson solve or direct integration?
ANSWER: POISSON SOLVE — already optimal.

The method solves nabla^2 h = div(N) via SOR (same as the transport map solve).
No path-dependent integration. No systematic drift.
An FFT-based solver would be faster (O(n log n) vs O(n^2)) but yield
mathematically equivalent results.

IMPROVEMENT POSSIBLE: No, for correctness.
SPEED IMPROVEMENT POSSIBLE: Yes (FFT Poisson) — would save a few seconds
at 128px but not meaningful at HYPER scale. At 512px+ it could matter.
